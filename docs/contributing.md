# Contributing to TUIKit

We love contributions! Here's how you can help.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR-USERNAME/SwiftTUI.git`
3. Create a branch: `git checkout -b feature/my-feature`

## Development

Build the project:

```bash
swift build
```

Run tests:

```bash
swift test
```

Run the example application:

```bash
swift run TUIKitExample
```

## Making Changes

### Code Style

- Use meaningful variable and function names
- Follow Swift conventions
- Add documentation comments for public APIs
- Keep functions small and focused

### Documentation

When adding new features:
1. Add inline documentation comments
2. Update relevant documentation files
3. Add examples if applicable

## Testing

Write tests for new functionality:

```bash
swift test
```

All tests must pass before submitting a pull request.

## Submitting Changes

1. Commit your changes: `git commit -am 'Add feature'`
2. Push to your branch: `git push origin feature/my-feature`
3. Open a Pull Request with a clear description

## Pull Request Process

1. Update documentation if needed
2. Add or update tests for new functionality
3. Ensure all tests pass
4. Get approval from maintainers
5. Merge your PR!

## Questions?

- Open an issue on GitHub
- Join our discussions
- Contact the maintainers
